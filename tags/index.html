Tags in HTML

In HTML, tags are special keywords written inside angle brackets <> that define the structure and meaning of content on a webpage. Most tags come in pairs, such as <p> (opening) and </p> (closing), with the content placed in between, while some tags like <br> and <img> are self-closing. Tags can also have attributes (like href, src, id, or class) that provide extra information about the element. For example, <a href="https://google.com" >Google</a> creates a hyperlink. Different types of tags serve different purposes: structural tags (<html>, <head>, <body>), text formatting tags (<h1>â€“<h6>, <p>, <b>, <i>), media tags (<img>, <video>, <audio>), form tags (<form>, <input>, <button>), and semantic tags (<header>, <footer>, <nav>, <section>). Together, these tags allow developers to build web pages that are structured, interactive, and meaningful for both browsers and users.

File path in HTML

In HTML, a file path is a way to tell the browser where to find a file, such as an image, CSS stylesheet, JavaScript file, or other resources that are linked to a webpage. There are two main types of file paths: absolute paths and relative paths. An absolute path provides the full URL to the file, starting from the domain name, for example, https://example.com/images/photo.jpg. This type of path points to a fixed location on the web and can be accessed from anywhere. On the other hand, a relative path specifies the fileâ€™s location in relation to the current HTML file, making it easier to manage files within the same project. For example, images/photo.jpg points to a file inside the images folder located in the same directory as the HTML file, while ../images/photo.jpg points to a file in the images folder located one level above the current directory. Using correct file paths ensures that resources are loaded properly, allowing the webpage to display images, apply styles, and execute scripts as intended.

Two types of file-path in html

  • Relative file path
  • Absolute file path



Relative & Absolute file path!

1. Relative File Path

A relative file path in HTML specifies the location of a file in relation to the current HTML document. It does not include the full URL but instead uses the folder structure of your project to locate files. For example, if an image is in the same folder as the HTML file, you can simply use image.jpg as the path. If the image is in a subfolder called images, the path would be images/image.jpg. Relative paths are useful when moving a website between servers because they do not depend on the full domain name.

2. Absolute File Path

An absolute file path in HTML provides the full URL or complete path to a resource, including the domain name or root directory. For example, an image hosted on a website could be linked using https://example.com/images/image.jpg. Absolute paths are helpful when linking to files hosted on other servers or external websites. However, they are less flexible for moving files within your own project, because the full path must remain the same.
